Abstraction:
----------------------
This principle helps us hide unnecessary internal details and show only the important part.
Abstraction came to:
-> hide unnecessary details
-> show only essential features
-> reduce complexity 

ABSTRACTION CAME TO REDUCE THE COMPLEXITY. ->key reason

Better definition:
-------------------
Abstraction means showing what an object does, not how it does it internally

Abstraction = what to do, not how it is done.

Abstraction is useful because:
---------------------
-> it reduces complexity
-> it improves readability
-> it makes code easier to use
-> it hides unnecessary implementation
-> it allows focus on essential behaviour.

In Java, abstraction is mainly achieved using:
1. abstract class
2. abstract method

Later there are also interfaces, but first let us clearly understand abstract class.

Abstract class?
-----------------------
An abstract class is a class that cannot be instantiated directly and may
contain an abstract methods.

definition:
