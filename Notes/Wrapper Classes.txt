Wrapper Classes:
------------------
A wrapper class is a class that wraps a primitive data type into an object.
-> Wrapper class means object form of a primitive data type.

Best memory Line:
---------------------
Primitive = raw Value
Wrapper class = object form of that value

Primitive types and Wrapper classes:
--------------------------------------------
Primitive Type                 Wrapper Class
-----------------              -----------------
byte                                 Byte
short                                Short
int                                  Integer
long                                 Long
float                                Float
double                               Double
char                                 Character
boolean                              Boolean

Why primitive types are not enough?
---------------------------------
-> they are not objects
->they cannot be used directly in collections
-> they cannot store null
-> java API's often expects objects
 so wrapper classes solve this problem

Primitive vs wrapper
--------------------------
Primitive:
--------------
->stores value directly
->not an object
->faster
-> less memory
-> cannot store null
Wrapper:
-------------
-> object representation
->stores value inside object 
->can use methods  
-> can store null 
-> can be used where object is required.

valueOf() method:
------------------------
valueOf() converts primitive or string into wrapper Objects

public class ValueOfMethod {
    public static void main(String[] args) {
        Integer a = Integer.valueOf(10);
        Double b = Double.valueOf("25.5");
    }
}

parseInt() and parseDouble():
-----------------------------
These methods convert string into primitive values

public class ParseIntDouble {
    public static void main(String[] args) {
        int a = Integer.parseInt("100");
        double b = Double.parseDouble("45.8");
        System.out.println(a+10);
        System.out.println(b+5);
    }
}

Wrapper classes and null:
---------------------------
Primitive types cannot store null
Wrapper classes can store null , because those wrapper classes are object reference, so it can be null.

compareTo():
---------------
This method used to compare the objects and give output according to that.
if 1st num > 2nd num -> it gives positive value (1)
if 1st num <2nd num -> it gives negative value. (-1)
if num1 = num2 -> it gives 0

eg:
-----
Integer a = 10;
Integer b = 20;

Scenario 1:
-----------
a.compareTo(b) -> -ve value

Scenario 2:
------------
b.compareTo(a) -> +ve value

Scenario 3:
----------
b.compareTo(20) -> 0


public class CompareTo {
    public static void main(String[] args) {
        Integer a=100;
        Integer b = 50;
        System.out.println(a.compareTo(b));
        System.out.println(b.compareTo(a));
        System.out.println(a.compareTo(100));   
    }
}

What is a boxing?
-------------------------
If we convert primitive value into wrapper object manually,. that IS CALLED BOXING.

But writing valueOf() again and again is lengthy.
So java automatically does this conversion for us.
That automatic conversion is called "AUTOBOXING".


Definition:
--------------
Autoboxing is the automatic conversion of primitive data type into it's corresponding wrapper object.

public class Autoboxing {
    public static void main(String[] args){
         int a = 10;
       Integer b = a;  //autoboxing
        System.out.println(a);
        System.out.println(b);
    }
}
              converts to
primitive --------------------> Wrapper Object

UnBoxing:
--------------
Unboxing is the conversion of wrapper object into its corresponding primitive data type.

what is auto-unboxing:
-----------------------------
When java does this conversion automatically, it is called auto-unboxing.
                 Auto-unboxing conversion
Wrapper Object -----------------------------> Primitive value.

public class AutoUnBoxing {
    public static void main(String[] args){
        Integer obj = 20;
        int a = obj;
        System.out.println(obj);
        System.out.println(a);
    }
}
